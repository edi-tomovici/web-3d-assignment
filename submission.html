This project does not use VRML or X3D. The 3D models were built in Blender and exported as .glb (glTF binary) files, loaded with Three.js via the GLTFLoader. The .glb files and the original .blend source files are included in the GitHub repository linked above, inside the models/ and blender/ folders respectively.

4. Deeper Understanding Statement

Beyond the lab work, this project demonstrates the following:

  • Three.js with ES module import maps: the labs covered X3DOM, but I chose Three.js because it integrates better with Blender via glTF and has a much larger community. I loaded it through an import map so that the addons (GLTFLoader, OrbitControls) could be imported using clean ES module syntax.
  • Three-point lighting setup: I used an ambient light combined with a directional key light to give the cans depth and avoid flat shading. The directional light can be toggled by the user from the interface.
  • Automatic model centring: rather than hard-coding positions, the loader uses THREE.Box3 to compute the model's bounding box and reposition it so its centre sits at the world origin. This works for any model regardless of how it was built in Blender.
  • Wireframe persistence across model swaps: swapping a model loads it with fresh materials, so the wireframe flag has to be re-applied inside the loader callback rather than just on the button click. This was the trickiest bug to track down because of the async timing.
  • Version control with GitHub: the codebase is managed in a public Git repository so the development history is visible.

Feedback from testing: I tested the app in Chrome and Opera GX during week 11. The site is fully functional: all three models load, the wireframe and light toggles work and persist correctly when switching cans, the spin animation runs for two seconds and stops cleanly, and the description text updates with each model. The console is clean of errors on the live ITS server. One limitation noted during testing is that the 3D canvas itself is mouse-only — keyboard rotation would be a useful accessibility addition with more time.
