About Energy HQ

How this site was built

The Project

Energy HQ is a 3D web app I created for my third year Web 3D Applications module. The assignment required at least 3 interactive 3D models. I picked energy drinks because their can shapes were appropriate for a Blender beginner.

Design Choices

I chose a light grey background to help the cans stand out, and used red accents as a nod to the Red Bull branding. The 3D viewer is placed in the center, with controls below it. The description card has a red stripe on the left to catch attention without taking over the page.

The 3D Models

I made all three cans in Blender 5, starting with a basic cylinder shape. I used smooth shading to hide the low number of polygons. For textures, I used UV unwrapping with cylinder projection, which was the trickiest part and took the most trial and error.

The Web App

The app is built with plain HTML, CSS, and JS and uses Three.js via an ES module import map. I picked Three.js over X3DOM because it has a larger community and works better with Blender via glTF.

Features:

  • Switch between three energy drink models.
  • OrbitControls for camera rotation, pan, and zoom.
  • Wireframe toggle that continues across model swaps.
  • Directional light toggle.
  • Click-triggered spin animation.
  • Description text that updates with each model.

Challenges

While building the app, the trickiest bug was getting the wireframe toggle to persist when switching models, new models loaded with fresh materials, so I had to apply the wireframe state inside the loader callback rather than just on the button click. The async timing took a couple of tries to get right.

Testing

I tested the app in Chrome and Opera GX on Windows. All buttons work as expected, the wireframe toggle remains active when switching models, the spin animation runs for 2 seconds and stops smoothly, and the console shows clean load logs for all 3 models, with no errors.

Accessibility

I considered the following when building the page:

  • Used semantic HTML tags (<header>, <main>, <section>, <footer>) so screen readers can understand the page structure.
  • Set lang="en-GB" on the html tag so screen readers use the right pronunciation.
  • One limitation: the 3D canvas itself is mouse-only at the moment. Keyboard rotation would be something to add with more time.
